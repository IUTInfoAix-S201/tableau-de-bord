Comment lire ce tableau

  • Voyant qualité de revue (par contributeur) : vraies revues (commentaires, changements demandés), engagement léger, approbations à vide (tampon), pas encore de revue.
  • Taux de contribution : votre part du travail de l'équipe, calculée comme une moyenne pondérée de vos contributions sur six dimensions : PR mergées 25 %, lignes ajoutées 25 %, contribution par feature 25 %, issues fermées 10 %, revues données 10 %, travail en cours 5 %. La contribution par feature = la somme de vos parts dans le code de production (src/main) de chacun des 8 écrans : fiable et non manipulable par l'activation en masse de tests. Elle remplace l'ancien « tests validés », retiré du classement car gonflé par les merges mal résolus (la même activation comptée dans plusieurs PR) ; le décompte de tests ne subsiste qu'au niveau de chaque PR dans le détail dépliable. Chaque dimension étant rapportée au total de l'équipe, les taux des membres d'une équipe somment à 100 %. Survolez la valeur (dans le détail dépliable) pour voir le calcul dimension par dimension.
  • Facteur d'effort : un coefficient applicable à la note pour tenir compte de l'investissement individuel. Dans une équipe de N personnes, on attend en moyenne 1/N du travail (25 % à quatre). Si vous contribuez au moins cette part idéale, le facteur vaut 1 (aucune pénalité). En dessous, il décroît proportionnellement : facteur = min(1, taux ÷ idéal). Exemple à quatre (idéal 25 %) : un taux de 12,5 % donne un facteur 0,5 ; un taux de 30 % donne 1.
  • MVP ✅ = toutes les fonctionnalités MUST (dont l'écran pivot Passage) terminées.
  • Équilibre du travail : répartition des commits entre les membres (entropie normalisée de Shannon), de 0 à 100 %. 100 % = chacun contribue également ; bas = concentré sur quelques-uns (un passager clandestin le fait baisser). Apparenté au bus factor (en infobulle), mais à résolution fine pour les petites équipes.
  • Badges (tableau par étudiant) : 🏗️ le plus de commits · 🔍 le plus de revues · 🚀 le plus de PR mergées · 💬 le plus de commentaires de revue · 🐝 polyvalent (au-dessus de la médiane sur code, PR, revues et issues) · 🤝 relit autant qu'il est relu · 🧐 vraies revues exigeantes · 🌟 le code le plus relu · 🛡️ le plus de changements demandés · 🗺️ le plus de features touchées · 🐘 la plus grosse PR · 🐿️ le plus de toutes petites PR · 🦆 approbations à vide · 👻 passager clandestin (< 5 % du travail de l'équipe). Survolez un badge pour le détail.

⚠️ Les voyants et indicateurs collaboratifs sont des repères heuristiques, indicatifs et non des notes. Les chiffres bruts sont toujours affichés pour que chacun puisse les interpréter.
